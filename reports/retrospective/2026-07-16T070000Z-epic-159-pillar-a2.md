## Friction Patterns

### FP-01: PowerShell weakened-port case-sensitivity defect class, discovered only in three escalating layers — operator-level, cmdlet-level, then an evaluator's own empirical sweep

- **Evidence:** Both new `.ps1` full-parity ports this feature (`spec-review-precheck.ps1`, T-004/#174; `domain-review-precheck.ps1`, T-003/#147) shipped with PowerShell's default case-**insensitive** comparison operators/cmdlets silently widening preconditions the `.sh` originals' always-case-sensitive `bash`/`sed`/`jq` reject — a threat class `security-spec.md` B2 ("weakened-port") had already named at spec time, yet no reviewer, the implementer, or the host caught a concrete instance before quality-gate:
  - **Layer 1 (operator-level, T-004 gate cycle 1, seq 0225, Critical):** `plugins/sdd-review-loop/scripts/spec-review-precheck.ps1:146`'s feature-slug check used `-notmatch` (case-insensitive), accepting `Test-Feature`/`NOFEATURE`/`Epic-159-Pillar-A2` the `.sh` original's `bash =~` always rejects — a replay-protection bypass on case-sensitive filesystems. Fixed `2a7eef5`: 1 site (line 146) + 1 more `-match`/`-notmatch` site (line 447, `--reset` directory scan) + 10 more `-eq`/`-ne`/`-notcontains` sites against JSON-sourced fields = **12 operator-level fixes in this one file**. The same evaluator *diagnosed* (did not edit, per its own amendment scope) the identical class in T-003's `domain-review-precheck.ps1`; the host routed the fix separately: `69cda39` found and fixed 2 genuine sites (`:137` status regex, `:255` `--reset` directory scan) plus 1 defensive hardening (`:256`).
  - **Layer 2 (cmdlet-level, a *second* independent evaluator cycle per file, Critical+Major):** the layer-1 sweep was operator-only and missed that PowerShell cmdlets carry their **own, independent** case-(in)sensitivity defaults. T-004 gate cycle 2 (seq 0228, Major): `spec-review-precheck.ps1:187`'s `Select-String` matched a `SPEC-REVIEW-STATUS:` header the `.sh` original's `sed` never matches — fixed `19cfb75`. T-003 gate cycle 2 (seq 0227, Critical + Major): `domain-review-precheck.ps1:135`'s `Select-String` (Critical) and `:118`'s `Get-ChildItem -Filter '*.md'` (Major — case-insensitive on macOS, the AC-017 CI matrix's own `macos-latest` runner, wrongly accepting a file literally named `card.MD`) — both fixed `b5eba54`.
  - **Layer 3 (evaluator's own empirical sweep, T-004 gate cycle 3, seq 0230, PASS):** the third and final evaluator cycle went beyond re-checking the two reported findings and empirically tested a cmdlet neither prior layer had named — `Select-Object -Unique` — confirming it, too, is case-sensitive by default, before declaring zero residual divergence.
  - Combined: **~14 genuine operator-level sites (12 + 2) plus 3 cmdlet-level sites (1 + 2)** across the two ports, discovered only by an isolated quality-gate evaluator across 2-3 of its own internal cycles per file — never by the task-review/spec-review reviewers who approved `tasks.md`'s own Risk Rationale citing "security-spec.md B2 weakened-port threat row" as a risk-classification justification, never by the implementer who wrote and self-tested both ports as "full-parity," and never by the host who generated the first round of gate evidence (`b1922b8`).
- **Frequency:** 2 of the feature's 4 tasks (T-003, T-004), each recurring across 2-3 independent evaluator cycles with byte-for-byte `.sh`-vs-`.ps1` parity re-verification logged at every layer.
- **Phase:** implementation (translation-layer authoring) + quality-gate (the only phase that caught it, and only after 2 of its own internal escalations).
- **Confidence:** High (recurs across 2 tasks, 3 escalating discovery layers each, all independently evidenced with exact file:line citations and live counterfactual regex/cmdlet probes).
- **Do Not Overfit:** every instance was caught and fixed cleanly — zero residual divergence at final cycle, zero task attempts beyond 1, zero review tickets (all findings closed inside the gate). This is not a failure of the quality-gate's own discipline; it is direct, measured proof of the multi-cycle gate's value: 2 reviewers (spec-review + task-review) + the implementer + the host all missed the same defect class the spec's own threat model named in advance, and only the independently isolated evaluator caught it — but even the evaluator needed a second, distinct sweep layer (cmdlets, not just operators) and a third confirmatory empirical pass before the class was fully closed. - **Frequency:** recurs across ≥2 retrospective periods (the prior `gh`-grep incident + this period's 27-finding instance) with 2 independent evidence types (the deterministic scan's own FAIL log and the fix commit's root-cause description), plus one within-feature near-miss that was correctly avoided (T-001).

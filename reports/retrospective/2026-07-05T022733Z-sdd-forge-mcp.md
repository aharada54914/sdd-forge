### FP-001: Review-gate precheck scripts contradict each other, forcing manual fallback

- **Evidence:** spec-review precheck validates absolute-path contracts including precheck/summary files while impl-review precheck requires relative paths and specs/-only with current hashes (mutually unsatisfiable — GitHub issue #61); task-review precheck false-positives on Risk/Workflow file-wide grep (`reports/task-review/sdd-forge-mcp/attempt-1/round-3/manual-precheck-note.md`); the quality gate then also ran manually under the same incompatibility (all 11 QG reports carry the issue-#61 note).
- **Frequency:** 4 workflow phases (spec-review, impl-review, task-review, quality-gate) across the whole feature
- **Phase:** review gates + quality gate
- **Confidence:** High
- **Do Not Overfit:** three independently implemented gate scripts plus the QG runner all exhibited the incompatibility; this is structural, not a single-task accident.

### FP-002: Evidence artifacts and measurement tooling disagree on schema

- **Evidence:** all 11 implementation reports lack `Task Attempt Count`/`Run ID`; all 11 QG reports use `Task ID:` without `Run ID:`, failing this skill's own artifact rule 3; `reports/runs/` was empty, so no attribution baseline existed. Every `N/A` in the Metrics table above is caused by this drift.
- **Frequency:** 22 artifacts across all 11 tasks
- **Phase:** implementation reporting + quality gate + retrospective (measurement)
- **Confidence:** High
- **Do Not Overfit:** the drift is uniform across every report author and task type in the period, and it degrades the retrospective's own deterministic counting — it is a measurement-pipeline property, not a task property.

### FP-003: Batch identity-ledger reservation goes stale (Low confidence — no WFI)

- **Confidence:** Low — recorded for future recurrence tracking only; must not create a WFI.

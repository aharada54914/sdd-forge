fail closed at the jq wall. The Outputs hash for the test file reflects the

## Summary

Added the domain-review gate for the sdd-domain plugin, replicating the
`spec-review-loop` launch-boundary / identity-ledger pattern exactly (per
this task's Must-Read instruction) with two new domain-specific reviewer
roles:

- `domain-reviewer-a` (strategic): context boundaries, relation patterns,
  event coverage, term uniqueness, aggregate-context ownership, and
  presence of the `Domain-Model-Status` field.
- `domain-reviewer-b` (tactical): invariant verifiability,
  transaction-boundary realism, god-aggregate / anemic-model risk, and
  lifecycle definition.
- `domain-review-loop/SKILL.md`: the orchestrator, identical AC-005
  aggregation rule to spec/impl/task-review-loop (PASS requires both
  reviewers free of Critical/Major FAIL; NEEDS_WORK before round 3; BLOCKED
  at round 3; round-3 Minor-only is PASS with nonzero `warningCount`), plus
  the AC-014 drift-detection precondition folded into its own precheck step
  (a distinct control from T-006's hook guard, which only rejects an
  agent-authored `Approved` write and says nothing about content changing
  *after* a human already approved).
- `domain-review-calibration.md`: the calibration reference both reviewers
  read first, scoping the gate to strategic/tactical domain-model soundness
  only (mirrors `spec-review-calibration.md`'s structure and purpose:
  Source Lessons, Gate Responsibility, Finding Evidence Gate, Severity
  Calibration, False-Positive Guard, Reproducibility).
- `plugins/sdd-domain/scripts/domain-review-precheck.sh`: the precheck and
  hash-binding script, analogous to `spec-review-precheck.sh` but simpler
  (no per-feature `specs/<feature>/` tree — it reviews the project-level
  `domain/` tree) and with the added AC-14 drift-detection logic.

1. `Should Match "never receive.*reviewer-a\.json|must never receive"`

2. `Test-DomainDrift`'s return value collapsed to `$null` for a zero-result

   output-stream unrolling pitfall already documented in this repository's
   own T-001 report and its Global Constraints), causing

   wrapping every call site in `@(...)`.
3. `Invoke-DomainReviewPrecheck`'s Windows-to-Unix path conversion used an
   invalid `-replace` scriptblock-evaluator form (not supported by
   Windows PowerShell 5.1's regex `-replace` operator) and its native
   `bash.exe` invocation, captured via `2>&1` under
   `$ErrorActionPreference = "Stop"`, promoted every nonzero-exit stderr
   line to a terminating `RemoteException` instead of returning it as data.
   Fixed the path conversion with explicit substring slicing
   (`C:\foo\bar` -> `/c/foo/bar`) and wrapped the native call with a
   scoped `$ErrorActionPreference = "Continue"` restore-on-`finally`.

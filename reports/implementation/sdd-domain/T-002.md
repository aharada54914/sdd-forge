## Outcome

Added `plugins/sdd-domain/skills/domain-interviewer/SKILL.md`, an internal
(`disable-model-invocation: true`, `user-invocable: false`) skill, plus its
seven adapted templates, that:

1. Accepts seed input in four forms: free text, a local Markdown path, an
   issue URL, or a `domain-reverse` (T-003) candidate seed handed off
   in-session. Local-path and URL failures halt with a plain-language error
   naming the failed seed and never invent content (AC-004).
2. Interviews through the seven canonical stages -- Domain Story, Event
   Storming, Ubiquitous Language, Context Map, Domain Model (aggregates),
   Domain Message Flow, C4 Container -- one topic at a time, checkpointing
   each stage's artifact to its canonical path before the next stage's
   questions begin.
3. Documents a stage-detection resume algorithm: at the start of every run,
   check the seven canonical paths in stage order and resume at the first
   missing one; an interruption after stage 3 resumes at stage 4 without
   re-touching stages 1-3.
4. Documents `domain/domain-contract.json` regeneration after every stage
   checkpoint (not only the final one), validated against
   `contracts/domain-contract.v1.schema.json` before the stage is reported
   complete.
5. Ships its own adapted C4-container template under
   `domain-interviewer/templates/`, consulted at build time (this task) from
   `plugins/sdd-bootstrap/skills/sdd-bootstrap-interviewer/templates/c4-container.template.md`
   but never read across the plugin boundary at runtime, per the feature's
   Global Constraints.

## Consuming domain-reverse's candidate seed

SKILL.md's "Consuming a domain-reverse Candidate Seed" section restates
(does not read at runtime) the exact five-section contract documented in
`plugins/sdd-domain/skills/domain-reverse/SKILL.md`'s "Candidate Seed
Shape": `candidate_contexts`, `candidate_terms`, `candidate_event_hints`,
`candidate_aggregate_hints`, `candidate_open_questions`. A seed is
identified by its `Seed-Source: reverse (investigate-codebase, <feature>)`
header. Each bucket pre-populates a specific stage (contexts -> stages 1
and 4; terms -> stage 3; event hints -> stage 2; aggregate hints -> stage
5; open questions -> whichever stage they bear on) as a human-revisable
proposal, never as an unreviewed final answer -- every candidate's
`INV-NNN` evidence is carried forward into the checkpointed artifact so a
reviewer can trace a term or aggregate back to the originating code.

   nonexistent section name).
3. **Deciding how strictly to interpret "own copy... adapted"** for the C4
   container template: the sdd-bootstrap original
   (`plugins/sdd-bootstrap/skills/sdd-bootstrap-interviewer/templates/c4-container.template.md`)
   is generic enough that a literal copy would already satisfy the Global
   Constraints' letter. Chose to adapt further: added a "Context-to-Container
   Mapping" section tying containers back to this plugin's own Context Map
   stage (a domain-interviewer-specific need the generic template has no
   reason to carry), so the "own copy, adapted" language in `tasks.md` and
   `design.md`'s Assumptions is substantively true, not just
   path-substantively true.
4. **PS1 hygiene technically applies to `.ps1` files, not `.md` template
   files** -- `tasks.md`'s Global Constraints PS1-hygiene rule is scoped to
   `.ps1` files (BOM-less ANSI interpretation risk). The seven templates
   are `.md` files, not `.ps1`, so that specific corruption risk does not
   apply to them directly. However, AC-013 requires templates to be
   English, and this repository's only precedent for machine-checking
   "English" is the non-ASCII-literal convention used elsewhere for PS1
   hygiene; reused that same practical signal for
   `template-language.Tests.ps1`'s conformance check rather than inventing
   a different, unprecedented "is this English" heuristic. This is a
   judgment call, flagged here for the orchestrator's awareness: a
   template could theoretically contain a stray non-ASCII character that
   is still valid, uncontroversial English-adjacent notation (e.g. a
   micro sign or a smart quote pasted from a word processor) and this test
   would flag it as a failure requiring an ASCII substitute, which is the
   conservative, safe-by-default choice consistent with this repo's
   existing PS1-hygiene precedent.

One intermediate failure was found and fixed before the green run above:
`$script:twoCtxContract` is a plain PowerShell `[hashtable]` literal (built
with `@{ }`, not round-tripped through `ConvertTo-Json`/`ConvertFrom-Json`),
so its nested `relations[0]` value is itself a `[hashtable]`, not a
`PSCustomObject`. The `Get-PropSafe` helper (built for `PSCustomObject`'s
`.PSObject.Properties`) silently returned `$null` against it under
`Set-StrictMode -Version Latest`. Fixed by indexing the hashtable directly
(`$relation["from"]`, `$relation["pattern"]`) instead of routing through
`Get-PropSafe`, which remains correctly used everywhere a fixture is
round-tripped through JSON (the `domain-contract.json` fixtures, which
really do parse into `PSCustomObject`).

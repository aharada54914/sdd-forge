immediately after the existing CHANGELOG-heading check
(`scripts/bump-version.sh:38-42`, unedited) and before every mutation step

`sed -i` at `:71`): a `for` loop over `tests/loop-consistency.tests.sh` and
`tests/loop-inventory.tests.sh`, invoked via `"${ROOT}/${suite}"`, that
captures each suite's output to a `mktemp` file and `exit 1`s

either suite exits non-zero. No environment-variable or CLI-flag bypass
exists. Authored `tests/bump-version-gate.tests.sh` / `.ps1`
(TEST-001..TEST-006) locking this behavior against a tar-copy +
local-`git init` fixture-repository copy (never the real repository),
following `tests/repository-release-validation.tests.sh:9-16`'s tar-copy
precedent extended with a git baseline so `git status --porcelain` becomes
a meaningful zero-mutation proof. Registered both suites in
`tests/run-all.sh`/`.ps1` and `.github/workflows/test.yml` (bash-then-pwsh
step pairing, following the loop-suite/`hitl-wfi-terminal` precedent — no
`if: runner.os != 'Windows'` guard on the bash step, matching that exact
precedent). Commit B: CREATEd the `CHANGELOG.md` `## Unreleased` entry
citing #148 (bump-version.sh leg; T-002 will append the release.yml leg to
the same entry) and CREATEd `docs/contributor/release-runbook.md` with the
bump-version.sh section and the REQ-004 cross-host degradation note.
Acceptance-first order was followed throughout (see Test Evidence).

Two implementation-time findings, both disclosed as deviations:

1. **Pre-existing, out-of-scope portability gap** (discovered via this
   task's own TDD, not introduced by it): `scripts/bump-version.sh`'s
   mutation section (unedited by this task, design.md API/Contract Plan)
   uses GNU-only `sed -i "<script>" "<file>"` syntax, which is incompatible
   with BSD/macOS `sed` (the `-i` flag requires a separate extension
   argument on BSD sed; without one, BSD sed misparses the script text as
   the extension and the file path as the script). This has apparently
   never been exercised in an automated way before this task (grep over
   `tests/*.tests.sh` and `.github/workflows/*.yml` before this task found
   zero references to `bump-version`). `TEST-001`'s mutation-success
   assertions degrade to a named SKIP (never a silent pass) on such hosts
   via a `sed --version` capability probe (GNU sed accepts `--version`; BSD
   sed does not) — `TEST-002..006` are unaffected, since their red paths
   never reach the mutation section and `TEST-004..006` never invoke the
   real script's mutation logic at all. Not fixed (design.md explicitly
   marks `scripts/bump-version.sh:51-70`/now `:57-76` as unedited by this
   task); recommend a follow-up issue.
2. A bug in this task's own new suite, found and fixed during
   implementation: `build_fixture()` appended to `CLEANUP_ROOTS` from
   inside a function invoked via command substitution (a subshell), so the
   array mutation never propagated to the parent shell. Combined with the
   `"${CLEANUP_ROOTS[@]:-}"` empty-array-safety idiom (which substitutes a
   single empty-string element when the array is genuinely empty), the
   EXIT trap's cleanup loop always executed `[[ -n '' ]]` at least once,
   which is false — and under `set -e` inside an EXIT trap, this silently
   forced the suite's process exit code to 1 regardless of actual
   pass/fail counts. Fixed by moving the `CLEANUP_ROOTS` append to each
   calling test function's own (non-subshell) scope. Verified: the suite
   now exits 0 exactly when `fail=0` (see Test Evidence).

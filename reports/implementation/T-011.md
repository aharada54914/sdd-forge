## Findings (the dogfood did its job)

### Finding 1 — `check-contract` medium tier is calibrated for compiled application code
Running `check-contract` on T-011's own contract fails:

```
risk medium requires check 'build' to be required:true
risk medium requires check 'lint' to be required:true
risk medium requires check 'typecheck' to be required:true
```

This repository is shell + Markdown + JSON; it has no lint/typecheck/build toolchain, and T-011
ships a JSON spec artifact plus gate-run logs. The medium-tier required-set
(`{lint, typecheck, build, unit-tests, acceptance-tests, regression, placeholder-scan,
task-state-check}`) forces `lint`/`typecheck`/`build` to `required:true`, which cannot be
satisfied here **without fabricating evidence** — which the core principle forbids. The checks
are therefore honestly waived (`required:false` + `waiver_reason`) and the gate correctly reports
the mismatch. This is why prior tasks T-002..T-010 never carried full contracts either.

**Recommendation (follow-up):** add a per-repo / per-task *check-applicability* concept so
`lint`/`typecheck`/`build` can be declared N/A when no toolchain exists (e.g. an allow-waiver list
keyed off a repo `stack` descriptor).

**RESOLVED in this follow-up (T-012):** `check-contract` Pass 4 now honors an optional `stack`
contract field. On a non-code stack (`shell`/`docs`/`spec`), `lint`/`typecheck`/`build` are
waivable (`required:false` + `waiver_reason`); all test/trace/placeholder/task-state checks stay
mandatory so a code task cannot abuse it. T-011's contract declares `stack: docs`, now passes
`check-contract`, and its evidence bundle validates. Mirrored in `.sh` + `.ps1`, tested
(`tests/gates.tests.sh` T-012.1–7, `tests/scripts.tests.ps1`), documented in `risk-gate-matrix.md`.

### Finding 2 — AC-009 wording is stale vs the shipped M-04 decision
`acceptance-tests.md` AC-009 says the agent role tomls "declare `model`". The shipped design
(capability-matrix M-04) intentionally does **not** pin a `model` key in those role tomls
(unconfirmed schema key -> "malformed agent role" rejection risk); cost-aware routing is a runtime
control (`--model`/`--effort`). Evidence: `T-009.presence.log`. The traceability link for REQ-010
records this supersession in its `note`. **Recommendation:** correct AC-009 wording to match M-04
(a spec-change that should carry a recorded rationale per REQ-005).

## Summary

Added a shared `upsert_mcp_json` helper (Node `-e` one-liner, Node >= 20 guaranteed
by the MCP gate) plus `register_cursor_mcp` / `register_vscode_mcp` functions to
`install.sh`, mirroring the existing `register_claude_mcp` / `register_codex_mcp`
conventions (function style, `echo ÔÇª >&2` notices, bash 3.2 compatibility,
test-overridable paths). VS Code's user-profile directory is resolved per-OS via
`uname` (macOS `~/Library/Application Support/Code/User`, Linux `~/.config/Code/User`).
Two new env overrides ÔÇö `SDD_CURSOR_DIR` and `SDD_VSCODE_USER_DIR` ÔÇö let the test
harness isolate the client config locations (mirroring the existing `SDD_CODEX_HOME`
override). The helper performs a key-based upsert that preserves all other server
entries and unknown top-level keys, emits stable 2-space `JSON.stringify` output
(byte-stable on re-run), skips with a notice when the client config directory is
absent (never creating the client dir), and refuses to overwrite a present-but-invalid
JSON file (error notice, installer continues with the other client). Both functions
are wired into `install_mcp_servers` behind the same `MCP_NODE_OK` / `--skip-mcp` /
`--mcp` gate as Claude/Codex (Cursor under `--target All`; VS Code under
`--target All|Copilot`).

## Working Notes

No delegated investigations. All work performed directly against the immutable
snapshot specs (design.md, security-spec.md, acceptance-tests.md, traceability.md)
and the live `install.sh` / `tests/install.tests.sh`. Notable safety decision:
because the existing harness scenarios run the installer with `--target All` and
real Node >= 20 on PATH, adding Cursor/VS Code registration would have made every
pre-existing scenario write to the developer's REAL `~/.cursor/mcp.json` and
VS Code user-profile `mcp.json`. The harness therefore exports isolated
`SDD_CURSOR_DIR` / `SDD_VSCODE_USER_DIR` defaults (pointing at non-existent
directories inside a temp root, cleaned by the EXIT trap) before any scenario
runs, mirroring the `SDD_CODEX_HOME` isolation pattern. Target scoping decision:
Cursor has no dedicated `--target` value, so it registers under `All` only;
VS Code (Copilot MCP) registers under `All` and `Copilot`. Note: the working
tree also contains a pre-existing orchestrator edit to `specs/local-env-mcp/tasks.md`
(T-007 Status ÔåÆ In Progress) that is not part of this change.

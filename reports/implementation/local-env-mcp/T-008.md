## Summary

Brought `install.ps1` to behavioral parity with the completed `install.sh`
(T-006/T-007 reference). Extended the `-Mcp` parameter's `ValidateSet` and
default to both `sdd-forge-mcp` and `local-env-mcp`, so a default install places
and registers BOTH MCPs while single-MCP selection is honored in either
direction. Added a shared `Update-McpJson` helper that shells out to the SAME
Node `-e` one-liner `install.sh` uses (Node >= 20 guaranteed by the existing
`Test-NodeVersionOk` / `McpNodeOk` gate) — this yields byte-identical 2-space
JSON output cross-platform, so re-runs are byte-stable and match the sh result.
Added `Register-CursorMcp` (`mcpServers.<name>`, `SDD_CURSOR_DIR` override) and
`Register-VSCodeMcp` (`servers.<name>` + `type: "stdio"`; per-OS user-profile
path — Windows `%APPDATA%\Code\User`, macOS
`~/Library/Application Support/Code/User`, Linux `~/.config/Code/User`;
`SDD_VSCODE_USER_DIR` override). Both are wired into `Register-McpServers` behind
the same `McpNodeOk` / `-SkipMcp` / `-Mcp` gate as Claude/Codex (Cursor under
`All`; VS Code under `All`/`Copilot`). Fail-safes match `install.sh` and
security-spec B3: an absent client directory is skipped with a notice and never
created; a present-but-invalid JSON file is left byte-identical with an error
notice (non-terminating, so the other client still registers and the installer
does not abort).

## Working Notes

No delegated investigations. All work performed directly against the immutable
snapshot (specs, AGENTS.md, report template) and the reference `install.sh` /
`tests/install.tests.sh` plus the live `install.ps1` / `tests/install.tests.ps1`.
Notable decisions: (1) mirrored the sh harness's global `SDD_CURSOR_DIR` /
`SDD_VSCODE_USER_DIR` isolation into the ps1 harness — without it, adding
Cursor/VS Code registration would have made every pre-existing `-Target All`
scenario write to the developer's REAL client configs; the isolated defaults point
at non-existent dirs in a temp root cleaned at suite end. (2) `Update-McpJson`
shells out to the identical Node `-e` script `install.sh` uses (rather than a
PowerShell-native `ConvertTo-Json`) so the emitted JSON is byte-identical to the
sh output and re-runs are provably idempotent (Node >= 20 is guaranteed by the
`McpNodeOk` gate). (3) Corrupt-JSON fail-safe uses `Write-Error -ErrorAction
Continue` so the notice does not terminate under the script's
`$ErrorActionPreference = "Stop"`; the native `node` exit code 3 populates
`$LASTEXITCODE` without throwing, so the other client still registers and the
installer exits zero. (4) VS Code per-OS path detection is self-contained (works
on both Windows PowerShell 5.1 Desktop and PowerShell 7+ cross-platform) rather
than relying on the test file's `$isWindowsPlatform`. Note: the working tree also
contains a pre-existing orchestrator edit to `docs/workflow-improvements/WFI-001.md`
that is unrelated to this change.

## Target

Feature `local-env-mcp`, task T-009: extend `uninstall.sh` and `uninstall.ps1`
so that uninstalling removes, for every installer-managed MCP (`sdd-forge-mcp`
and `local-env-mcp`): the placed payload under `<install-root>/mcp/<name>/`, the
Claude registration (`claude mcp remove <name>`), the Codex `config.toml` marker
block, and — new in this task — only the `mcpServers.<name>` keys in the Cursor
`~/.cursor/mcp.json` and the `servers.<name>` keys in the VS Code user-profile

JSON is left unmodified with an error notice; an absent file/dir is a silent skip.
Satisfies AC-012 / REQ-010 (security-spec.md B3, the "installer 以外が書いた
エントリを uninstall が誤削除" STRIDE row).

## Summary

Extended both uninstallers to be symmetric with the final installer for every
installer-managed MCP (`sdd-forge-mcp` and `local-env-mcp`):

- Genericized the MCP selection from `sdd-forge-mcp` only to both names
  (`VALID_MCPS` / `[ValidateSet(...)]` and the default list), so payload
  removal, `claude mcp remove`, and Codex `config.toml` marker-block stripping
  now cover both MCPs. These three paths were already generic over
  `MCP_SELECTION` / `$Mcp`, so widening the allowed set was sufficient.
- Added Cursor unregistration: deletes only `mcpServers.<name>` from
  `~/.cursor/mcp.json` (honoring `SDD_CURSOR_DIR`), participating in `--target
  All` (mirroring the installer, which has no dedicated Cursor target).
- Added VS Code unregistration: deletes only `servers.<name>` from the per-OS
  user-profile `mcp.json` (honoring `SDD_VSCODE_USER_DIR`), participating in
  `All` and `Copilot`.
- The JSON key deletion is the byte-for-byte inverse of the installer's
  `upsert_mcp_json` / `Update-McpJson`: the SAME Node one-liner (stable 2-space
  JSON + trailing newline, atomic temp-file rename) with a `delete` instead of
  an upsert. It preserves user-defined entries and unknown top-level keys.
- Fail-safes (security-spec B3): corrupt JSON → file left byte-identical, error
  notice, continue with the next client; absent file/dir → silent skip / notice
  (no error, no directory creation). Node is not a hard requirement — if `node`
  is absent, a one-time notice is printed and IDE-config removal is skipped
  while payload/Claude/Codex removals still run.

An initial defect (top-level `return` inside the Node `-e` script — illegal in
Node's eval context) was caught by the sh suite during Green and fixed by
switching the silent-skip paths to `process.exit(0)`, matching the installer's
own Node one-liner structure.

## Files Changed

- `uninstall.sh` — genericized `MCP_LIST` / `VALID_MCPS` to both MCPs; added
  `remove_mcp_json_keys` (Node-based key deletion), `mcp_json_node_available`
  (one-time Node-absent notice), `unregister_cursor_mcp`, `unregister_vscode_mcp`;
  wired Cursor (All) and VS Code (All/Copilot) into the unregistration flow;
  updated usage text and environment-variable docs.
- `uninstall.ps1` — widened `-Mcp` `ValidateSet`/default to both MCPs; added
  `Test-McpJsonNodeAvailable`, `Remove-McpJsonKeys` (same Node one-liner as sh),
  `Unregister-CursorMcp`, `Unregister-VSCodeMcp` (per-OS VS Code path +
  overrides); wired Cursor (All) and VS Code (All/Copilot) into the flow.

## Tests Added Or Updated

Appended four scenarios (o, p, q, r) to each suite in the existing harness style,
using `SDD_CURSOR_DIR` / `SDD_VSCODE_USER_DIR` overrides so the real user profile
is never touched:

- **(o)** Cursor + VS Code: managed keys for BOTH MCPs deleted; user-defined
  `my-server` key and unknown top-level keys (`telemetry` / `inputs`) survive;
  result is still valid JSON.
- **(p)** Both MCPs fully removed: both payload dirs gone, `claude mcp remove`
  invoked for each, both Codex marker blocks stripped, unrelated Codex section
  preserved.
- **(q)** Corrupt-JSON fail-safe: a corrupt Cursor `mcp.json` is byte-identical
  (SHA-256 compared) after uninstall, an invalid-JSON error notice is emitted,
  and the (valid) VS Code file is still cleaned (per-client continuation).
- **(r)** Absent Cursor/VS Code dirs → uninstall exits 0, install root still
  removed, and neither IDE directory is created.

- `tests/uninstall.tests.sh` — added `seed_installed_two_mcps` + scenarios o–r.
- `tests/uninstall.tests.ps1` — added `New-InstalledTwoMcpLayout` + scenarios o–r.

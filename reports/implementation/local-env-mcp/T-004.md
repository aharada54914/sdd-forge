### CI job for orchestrator to apply

Modeled on the existing `mcp-tests` job in the live `.github/workflows/test.yml`
(which exercises `mcp/sdd-forge-mcp`). This new job exercises
`mcp/local-env-mcp` with the same steps: `npm ci`, typecheck, test (which
includes the Inspector smoke test since `pretest` compiles
`tests/smoke/**`), and a dist-parity rebuild-and-diff check on Linux only.
Add this as a **new, separate job** (not merged into the existing
`mcp-tests` job, since `working-directory`/`cache-dependency-path` differ
per package) and add its job key to the `required-checks` job's `needs`
list if that job should gate merges (recommend doing so, matching how
`test` and `cli-hook-enforcement` currently gate).

```yaml
  # local-env-mcp-tests â€” local-env-mcp (mcp/local-env-mcp/) is a read-only
  # MCP server exposing local environment info (OS, toolchain versions, CLI
  # availability) with no execution capability (ADR-0003/ADR-0004 pattern,
  # same shape as sdd-forge-mcp). Exercised across the same 3-OS matrix as
  # sdd-forge-mcp's mcp-tests job.
  local-env-mcp-tests:
    name: local-env-mcp-tests
    strategy:
      fail-fast: false
      matrix:
        os: [windows-latest, macos-latest, ubuntu-latest]
    runs-on: ${{ matrix.os }}
    defaults:
      run:
        working-directory: mcp/local-env-mcp
    steps:
      - uses: actions/checkout@9c091bb21b7c1c1d1991bb908d89e4e9dddfe3e0 # v7.0.0

      - name: Set up Node.js
        uses: actions/setup-node@a0853c24544627f65ddf259abe73b1d18a591444 # v5.0.0
        with:
          node-version: "20"
          cache: npm
          cache-dependency-path: mcp/local-env-mcp/package-lock.json

      - name: Install dependencies
        run: npm ci

      - name: Type check
        run: npx tsc --noEmit

      - name: Run tests
        run: npm test

      # dist-parity (AC-008): rebuild the esbuild bundle and confirm the
      # working tree's committed dist/index.js is reproducible from source.
      # Runs once on Linux to keep the check OS-agnostic and fast; esbuild
      # output is deterministic across platforms for this bundle (verified
      # locally: two independent builds produced an identical SHA-256).
      - name: Rebuild dist and verify parity
        if: matrix.os == 'ubuntu-latest'
        run: |
          npm run build
          git diff --exit-code -- dist/

      - name: Audit production dependencies
        if: matrix.os == 'ubuntu-latest'
        run: npm audit --omit=dev --audit-level=high
```

If the orchestrator wants this job to block merges, add
`local-env-mcp-tests` to the `required-checks` job's `needs: [test,
cli-hook-enforcement]` list (making it `needs: [test,
cli-hook-enforcement, local-env-mcp-tests]`) and add a corresponding
result-check block mirroring the existing `test` / `cli-hook-enforcement`
checks in that job's `run:` script.

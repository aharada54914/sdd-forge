## Summary

Established the `mcp/local-env-mcp/` TypeScript project (same shape as
sdd-forge-mcp) and implemented the three T-001 source modules TDD (Red -> Green):

- `src/envelope.ts` — Result envelope verbatim-compatible with sdd-forge-mcp:
  `OkEnvelope`/`ErrorEnvelope`, the 7-code `ErrorCode` enum, and
  `ok()`/`err()`/`isOk()`/`isErr()` helpers. A compile-time test asserts the TS
  `ErrorCode` union equals the contract `error.code` enum (WFI-001 row 7).
- `src/allowlist.ts` — frozen `as const` table of exactly the 14 required CLIs
  (node, npm, pnpm, yarn, bun, deno, git, gh, python3, go, rustc, cargo, java,
  docker), each `{ name, command, args, versionStream }`. All use `--version`
  on stdout except `java` (`-version`, stderr). Commands are bare binary names
  (no path/shell metacharacters); no user input can mutate the table.
- `src/probe-engine.ts` — `execFile` ONLY (no shell). Per-probe timeout 2000 ms
  (SIGKILL on expiry), output capped at 8 KiB via `maxBuffer` (child killed on
  overflow), global concurrency limit 4 (bounded worker pool), version
  normalized to first line / trimmed / <= 200 chars, per-entry 60 s in-memory
  TTL cache. Every failure mode is a per-entry `ProbeResult`
  (`probeError: not-found | timeout | output-too-large | nonzero-exit |
  spawn-error`) — never a thrown / whole-response error. No user input reaches
  `command` or `args`; a test-only `pathOverride` prepends a fixture directory
  to PATH (fixtures shadow real binaries yet system utilities remain resolvable)
  without changing which fixed commands are run.

Tests were written first and captured failing (Red: pretest `tsc` fails with
module-not-found, exit 2), then all 19 pass after implementation (Green,
exit 0). AC-004 (timeout/output-cap/kill -> per-entry failure, contract-enum
probeError) and AC-006 (static check: 0 fs-write APIs, 0
`exec`/`spawn`/`fork`/`eval`/`shell:true`; `execFile` only) both hold.
`tsc --noEmit` is clean.

Two defects were found and fixed during Green via genuine test failures
(details in Working Notes): (1) the probe-engine's `pathOverride` originally
REPLACED PATH, which broke the slow/verbose shims (they could not find
`sleep`/`seq`), masking the timeout/output-cap behavior — fixed to PREPEND; and
(2) the static-check `\bspawn\b` pattern false-positived on the `"spawn-error"`
contract literal — tightened to match only call/member/import usage.

## Working Notes

No subagent delegations were used (single-context implementation). Two
investigations run directly during the Green phase, both driven by real,
failing tests (not speculative):

1. Purpose: diagnose why the `slowcli` (sleep 30) and `verbosecli` (>8 KiB)
   fixtures returned `available: true` instead of `timeout` /
   `output-too-large`. Result: root cause was the probe-engine's `pathOverride`
   REPLACING `process.env.PATH` with only the fixture directory. The shims
   invoke system utilities (`sleep`, `seq`, `printf`) that live in `/bin` and
   `/usr/bin`; with those off PATH, `sleep 30` failed silently and the shim fell
   through to its `echo`, exiting fast with output. Fix: prepend the fixture
   dir to the inherited PATH (`path.delimiter` join) so fixtures shadow real
   binaries while system utilities stay resolvable. This is also the more
   correct production semantic (never nuke the environment). Files examined:
   `src/probe-engine.ts`, `tests/error-paths/probe-engine.test.ts`; verified
   by a standalone `node execFile` reproduction.
2. Purpose: the AC-006 static check flagged `probe-engine.ts: found "spawn"`.
   Result: `\bspawn\b` matched the substring in the `"spawn-error"` contract
   `probeError` enum literal (a `-` is a word boundary). The file uses no
   `spawn` API. Fix: tightened the exec-API patterns to match only usage
   position — `spawn(`, `.spawn`, or `{ spawn }` — so legitimate contract
   string literals are not false-positived, while genuine API usage is still
   caught. Files examined: `tests/readonly/static-check.test.ts`,
   `src/probe-engine.ts`.

Cross-test cache pollution was also handled: the probe-engine TTL cache is
module-level, so `beforeEach(__clearProbeCache)` was added to keep results
order-independent (the concurrency test caches `cargo/slowcli`, which otherwise
poisoned the later `cargo/goodcli` cache-reuse assertion).

## Summary

`local-env-mcp` is now a first-class MCP in `install.sh` alongside
`sdd-forge-mcp`. The change was minimal because the MCP placement and
registration machinery was already generic over `MCP_SELECTION`:

- `MCP_LIST` default is now `sdd-forge-mcp,local-env-mcp` (both ship by
  default).
- `VALID_MCPS` is now `sdd-forge-mcp local-env-mcp` (both are accepted `--mcp`
  names; unknown names are still rejected).
- The `--mcp` usage/help text lists both names.
- The dead per-name constants `CODEX_MCP_MARKER_BEGIN` / `CODEX_MCP_MARKER_END`
  (which hardcoded the single `sdd-forge-mcp` name and were never referenced â€”
  `register_codex_mcp` already builds the marker pair per-MCP from the MCP
  name at runtime) were removed. Their block comment was replaced with a
  comment explaining the per-MCP marker approach.

No changes were needed to `mcp_selected()`, `place_mcp_servers()`,
`register_claude_mcp()`, `register_codex_mcp()`, `node_version_ok()`, or the
`MCP_NODE_OK` gate: each already iterates `MCP_SELECTION` and resolves payload
source / entry point / marker by MCP name, so both MCPs are handled by the same
code paths. `bash -n install.sh` passes; the change is bash 3.2 compatible (no
associative arrays, no `${var,,}`).

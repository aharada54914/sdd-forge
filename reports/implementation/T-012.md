## Why

T-011's dogfood surfaced Finding 1: `check-contract`'s tier minimum forces
`lint`/`typecheck`/`build` to `required:true` for medium+ tasks, but this repo (shell + Markdown +
JSON) has no compile toolchain — so the only ways to "pass" were to fabricate evidence (forbidden)
or never author a contract (what T-002..T-010 did). The fix lets a contract declare its `stack`.

## What changed

- **`plugins/sdd-quality-loop/scripts/check-contract.sh`** + **`.ps1`** (Pass 4): read optional
  `stack` (default/absent/empty = `code`). On a non-code stack (`shell`/`docs`/`spec`) the
  compile-oriented checks `{lint, typecheck, build}` may be `required:false` (their `waiver_reason`
  is still enforced by Pass 2/3). Every other tier check — `unit-tests`, `acceptance-tests`,
  `regression`, `requirement-traceability`, `placeholder-scan`, `task-state-check` — stays
  mandatory for ALL stacks, so a code task cannot use `stack` to skip its tests. Unknown `stack`
  fails (`contract stack is invalid: <value>`) and falls back to strict `code`.
- **`tests/gates.tests.sh`** T-012.1-7 and **`tests/scripts.tests.ps1`** (mirror): the new behavior
  + the abuse-prevention guards (docs cannot waive unit-tests; waiver still needs a reason; explicit
  `code` stays strict; invalid stack fails; works at high+tdd).
- **`plugins/sdd-quality-loop/references/risk-gate-matrix.md`**: new "Stack descriptor" section
  (the canonical source the gate cites).

## Outcome

Quality-gate now runs repository-wide workflow-state validation before scoped
task checks. Impl/Task prechecks run the canonical validator for their feature
before retaining all explicit predecessor, contract, risk, and graph checks.

Done-state report validation no longer searches every quality report by task ID.
It uses the feature-scoped `quality_report` path declared by the evidence
bundle and verifies that path, artifact hash, task ID, and PASS verdict. A
same-ID report from another feature cannot satisfy the task.

The round-1 quality review found that a caller-selected PASS report could still
belong to another feature. Evidence validation now requires bundle, contract,
and quality-report feature identity in both runtimes. Existing evidence reports
were migrated by adding their feature identity and regenerating their bundles.
The round-2 review then found an empty-feature bypass. Feature equality is now
enforced whenever either bundle or contract declares a feature, while genuine
legacy bundles remain compatible only when both omit it.
The final round-3 review found duplicate report identities and a PowerShell
case-folding mismatch. Reports now require exactly one Feature line and both
runtimes compare the slug case-sensitively.

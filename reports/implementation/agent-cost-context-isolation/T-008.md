## Summary

Added a closed, pinned 74-entry rollback inventory and matching Bash and
PowerShell transaction executables. Both implementations verify the baseline,
reviewed-release, and current hashes before writing; validate the isolated
1.4.0 baseline; retain a verified present/absent backup; restore that backup
after any partial apply failure; and verify every post-apply baseline state.

The isolated fixtures cover successful rollback, current-file tampering, dirty
worktrees, unknown contract properties, pre-apply validation failure, and
injected partial-apply failure for both runtimes. The canonical fixture also
verifies every inventory entry against pinned commit `7df7318` and confirms
that the restored release surfaces identify version 1.4.0.

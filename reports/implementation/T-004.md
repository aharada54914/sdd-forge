# Implementation Report: T-004

## Target

`specs/claude-workflow-compatibility/tasks.md#T-004` — real Claude validation
in CI and release verification without changing the supported SDD workflow.

## Summary

- Added a real-Claude-CLI manifest suite to the existing Windows, macOS, and
  Linux CI matrix. It validates all shipped Claude manifests and proves an
  intentionally invalid `agents` fixture fails.
- Added an opt-in release smoke with isolated HOME, host configuration, cache,
  Codex agent, and install roots. Its only standard output is a JSON result
  containing host CLI version, released plugin version, install root, reload
  status, command/agent discovery, and an explicit skip reason.
- The smoke registers and inspects both `sdd-bootstrap` and `sdd-ship`, retains
  the dependent `sdd-review-loop` review surfaces, and proves `run`,
  `spec-review-loop`, `spec-reviewer-a`, and `task-reviewer-a` are present.
  Claude uses its supported update/restart boundary before a fresh details
  process; Codex and Copilot use a fresh process boundary.
- A skipped smoke exits `3`, rather than returning a green release result. A
  hard registration or discovery failure exits `1`.
- Raised the affected `sdd-bootstrap`, `sdd-quality-loop`, and
  `sdd-review-loop` Claude/Codex/Copilot manifests plus both marketplace
  catalogs to `1.1.1`. Repository validation asserts all eleven release
  entries are present, consistent, and newer than `1.1.0`.

## Intended-Function Preservation

The release smoke verifies both public commands, not just the bootstrap path:
`sdd-bootstrap:run` and `sdd-ship:run`. It also verifies the independently
reviewed workflow's `sdd-review-loop:spec-review-loop` and its named reviewer
agents. Codex is exercised in an isolated `CODEX_HOME` with the actual Codex
CLI, so the Claude compatibility work cannot silently remove Codex discovery.

## Target

Feature: ci-mcp / Task T-002 (Risk: high, Required Workflow: tdd).
Implement a GET-only fetch wrapper `github-client.ts` (host fixed to
`api.github.com`, owner/repo percent-encoded as URL path segments, no HTTP
method other than GET ever issued) and a deterministic upstream-error
normalizer `error-normalizer.ts` per REQ-006's fixed status→code mapping
(401 -> `auth-missing`; 403 with a rate-limit indicator -> `rate-limited`;
403 without one -> `upstream-error`; 404 -> `not-found`; 429 -> `rate-limited`;
5xx / network failure -> `upstream-error`). The upstream response body must
never be copied into a returned envelope.

## Summary

Implemented TDD (Red -> Green):

- `src/github-client.ts` — `githubGet<T>()` and `githubGetText()` (the latter
  added for T-013's plain-text job-log endpoint). Host is a module-level
  constant (`https://api.github.com`); every request URL is built by
  percent-encoding each `pathSegments` element individually via
  `encodeURIComponent` and joining onto that fixed base, so no path segment
  can introduce a new scheme/host or escape into a different path. The
  injected `GithubFetch`/`GithubTextFetch` function types pin `init.method`
  to the literal `"GET"` — there is no parameter through which a caller could
  select POST/PUT/PATCH/DELETE. A token, when provided, is attached only as
  `Authorization: Bearer <token>` and never logged/echoed. On any non-2xx
  response the upstream body is never read; the response is handed to
  `error-normalizer.ts` and only the normalized envelope is returned.
- `src/error-normalizer.ts` — `normalizeUpstreamResponse()` implements the
  REQ-006 mapping exactly: 401 -> `auth-missing` (`details.status: 401`); 403
  with `x-ratelimit-remaining: 0` or a non-empty `retry-after` header ->
  `rate-limited` (with a derived non-sensitive ISO-8601
  `details.rateLimitResetAt` when computable, never a raw header value); 403
  without either indicator -> `upstream-error` (`details.status: 403`,
  explicitly never `path-denied`, which is reserved for local input-guard
  use); 404 -> `not-found`; 429 -> `rate-limited`; 5xx and any other
  unexpected non-2xx status -> `upstream-error`; 2xx -> `undefined` (caller
  treats as success). `normalizeNetworkError()` maps any network-level
  failure (DNS, connection refused, timeout, thrown fetch) to
  `upstream-error`. Error `details` never carry a token, an `Authorization`
  value, or a verbatim upstream header/body value — only derived, fixed-shape
  metadata.

Tests were written first and captured failing (Red: pretest `tsc` fails with
module-not-found against `src/github-client.js`/`src/error-normalizer.js`,
exit 2), then all 32 of this task's cases pass after implementation (Green,
exit 0). AC-010 (all upstream status/network-failure branches map
deterministically, upstream body never leaked) and AC-011 (`rate-limited`
details carry only a non-sensitive reset time, never a token) both hold.
`tsc --noEmit` is clean.

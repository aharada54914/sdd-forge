## Target

Feature: ci-mcp / Task T-009 (Risk: high, Required Workflow: tdd).
Extend `install.sh` / `install.ps1` `VALID_MCPS` and the default `MCP_LIST` to
include `ci-mcp` (bundled by default), route it through the existing
multi-MCP `--skip-mcp` / `--mcp <list>` selection, placement (`dist/*` +
`package.json`), and Claude / Codex / Cursor / VS Code registration paths
(already generalized to N MCPs by local-env-mcp), and print read-only PAT
environment-variable guidance without ever persisting a token value.

## Summary

- `install.sh` (modified) -- `VALID_MCPS`/default `MCP_LIST` += `ci-mcp`;
  usage/help text updated; new token-guidance notice printed in the
  top-level success section, gated on `mcp_selected "ci-mcp" && MCP_NODE_OK`.
- `install.ps1` (modified) -- same: `ValidateSet`/default array += `"ci-mcp"`;
  the token-guidance notice is printed from `Write-InstallSummary`, gated on
  `($Mcp -contains "ci-mcp") -and $script:McpNodeOk`, placed before the
  `-Target=FilesOnly` early return so it still fires when only files are
  placed.
- `tests/install.tests.sh` (modified) -- extended scenario (t) with ci-mcp
  placement/registration/token-guidance assertions, added scenario (v-ci)
  (`--mcp ci-mcp`), extended (v)/(w2)/(y)/(z)/(aa)/(ab) to the 3-MCP array.
- `tests/install.tests.ps1` (modified) -- ps1 parity for all of the above.

No new placement/registration *logic* was required: `install.sh`/`install.ps1`
already implement a name-list-driven generic path (established when
local-env-mcp was added as the 2nd MCP); adding `"ci-mcp"` to the
selection lists was sufficient for it to flow through every existing
client-registration function.

Red evidence (`T-009-red.txt`) discloses a genuine deviation: the source
edits and the new test scenarios were authored in the same working session
(not literal red-first commits). To capture GENUINE red evidence rather than
a fabricated transcript, `install.sh`/`install.ps1` were reverted to their
last-committed pre-ci-mcp content (`git show HEAD:install.sh > install.sh`,
non-destructive, ci-mcp-edited versions backed up first) while the new test
scenarios were kept in place, and both suites were re-run against that
reverted baseline -- 13 genuine `FAIL:` lines (dist not placed, `claude mcp
add` not invoked for ci-mcp, missing token-var guidance, Cursor/VS Code
upsert missing the managed entry, etc.), then the ci-mcp-extended source was
restored and the suite went green (35 passed / 0 failed for `install.tests.sh`).

   (c) verifies the file exists and its SHA-256 matches. Any gap
   (missing/hash-mismatch/out-of-root) appends to a gap list; if the list is
   non-empty after the scan, the gap list prints to stderr (the DECLARED
   path string only — never a resolved/joined candidate path, per
   security-spec.md B1's Information Disclosure mitigation) and the script
   exits 1 BEFORE the sanitize/write/print code is ever reached — a
   structural property (that code is simply unreachable on a gap), not a
   conditional guard (AC-015/AC-016/AC-032).

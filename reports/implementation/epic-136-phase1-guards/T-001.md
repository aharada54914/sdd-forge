## Summary

The live `sdd-hook-guard.ps1` had no R-10 protected-path table, no shell
write-target analysis, and no Impl-Review-Status forgery check: on a Windows
host without python3/node the dispatcher falls back to `.ps1`, letting an agent
overwrite gate scripts / hook config / critical tests or self-declare
`Impl-Review-Status: Passed`.

This change stages a corrected `.ps1` that ports, faithfully from the T-002
staged `.py`/`.js` twins:

1. `$ProtectedGateSuffixes` / `$ProtectedGatePluginJsonSuffixes` /
   `$ProtectedBasenames` and `Test-IsProtectedGateFile` (normpath-collapsing,
   case-insensitive suffix match) -- denies Edit/Write/MultiEdit writes to a
   protected gate file (Check 2a-R10).
2. The full shell write-target analysis: `Tokenize-ShellCommand`,
   `Test-SimpleShellCommandIsSafe`, `Test-ShellWriteTargetsAreSafe`, the
   read-only short-circuit, and the REQ-002 working-directory-aware pass
   (`Apply-CdTransition` / `Test-CwdWriteTargetIsProtected` /
   `Test-SegmentWriteHitsProtected` / `Test-ShellCwdWriteHitsProtected`) that
   closes the `cd <protected-dir> && rm <basename>` and `pushd` bypasses --
   denies Bash writes while allowing read-only shell over protected paths.
3. apply_patch protected-target denial (Check 2a-R10).
4. `Test-DesignMd` / `Test-ImplReviewVerdictExists` /
   `Test-ImplReviewStatusPassedIncreases` -- denies introducing
   `Impl-Review-Status: Passed` to `design.md` without a PASS /
   PASS-with-warnings `integrated-verdict.json` (Check 2e).

The existing kill-switch, SDD_SUDO C-02 protection, approval / WFI /
Second-Approval / domain-model guards, and agent-role guard are carried over
verbatim (logic unchanged); only their message constants were re-encoded.

ASCII-only source (AC-015) is achieved with an `Expand-Unicode` decoder: every
non-ASCII byte in the bilingual deny messages is written as a `\uXXXX` escape
and reconstructed at load, so the runtime message bytes stay byte-identical to
the twins (verified by round-trip) while the source file is pure ASCII with no
BOM. No test in the repo compares guard message bytes across twins
(`guard-parity.tests.sh` and `constant-parity.tests.sh` assert exit-code /
constant parity only), so this preserves decision parity, which is what AC-001
through AC-003 and the PowerShell Parity Note require.

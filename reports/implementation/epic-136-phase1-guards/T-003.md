## Summary

- Added `check-quality-gate-cycle-limit.sh` (bash, `set -euo pipefail`) and
  `.ps1` (ASCII-only, no BOM) implementing an internal contract:
  `check-quality-gate-cycle-limit <task-id> [reports-dir]`, default
  reports-dir `reports/quality-gate`.
- The scripts count files under reports-dir whose CONTENT references the task id
  with a WORD-BOUNDARY match (so `T-001` never matches `T-0010`). An absent
  directory counts zero. Count 0/1/2 prints `continue` and exits 0; count >= 3
  prints `Escalate-Human` and exits 1. An invalid task-id (not `^T-[0-9]{3}$`)
  is a usage error, exit 2.
- The `.sh` uses `grep -rlwF`; the `.ps1` mirrors the fixed word-boundary
  pattern already used in `check-task-state.ps1`
  (`"\b" + [regex]::Escape($TaskId) + "\b"`, matched `-CaseSensitive` for parity
  with grep).
- Wrote `tests/quality-gate-cycle-limit.tests.sh` RED-first (table-tested
  0/1/2/3/4, prefix collision, absent dir, invalid id, sh/ps1 parity, ASCII/BOM
  byte check), captured RED, then implemented to GREEN.
- Staged the complete corrected `ship/SKILL.md` under
  `specs/epic-136-phase1-guards/human-copy/ship-SKILL.md` with a one-hunk
  minimal diff (Step 4 item 5 only) and a `T-003.MANIFEST.sha256` line. The live
  protected `ship/SKILL.md` was NOT written by the agent (R-10 gate).

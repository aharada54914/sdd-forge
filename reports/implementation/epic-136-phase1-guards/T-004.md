## Workflow Walk-Through (behavioral trace of the new Step 4 contract)

1. Full track, task with `Risk: critical`, no `--verify`, no waiver → item 1
   forces `cross-model-verify` before the gate (was previously skipped without
   `--verify`).
2. Same task, `Cross-Model-Waiver:` present but NO co-located human
   `Approval: Approved` second-approver mark → waiver treated as ABSENT
   (fail-closed) → cross-model still required.
3. Same task, no cross-model run and no valid waiver → STOP before the quality
   gate with the L145 diagnostic naming `T-NNN`.
4. Same task offered to the lite track → Step 4 Lite track ineligibility
   pre-check rejects it with the L203/L204 diagnostic toward `--full`, so it
   never reaches a track lacking a cross-model step.
5. Non-critical task with `--verify` + `Cross-Model: enabled` → unchanged opt-in
   path still runs (preserved verbatim).

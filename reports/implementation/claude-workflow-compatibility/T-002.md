# Implementation Report: T-002

## Target

Establish the shared portable review-contract validation foundation for
`claude-workflow-compatibility`.

## Summary

Added a canonical `review-contract/v1` schema, a shared UTF-8 fixture corpus,
and equivalent Bash and PowerShell validators. Both validators deny malformed
slugs, noncanonical or nonpositive counters, inconsistent contracts, invalid
SHA-256 values, and report destinations outside the repository's `reports/`
directory before emitting canonical PASS JSON.

## Working Notes

- Independent second reviewer: Dewey. Initial P1 finding: PowerShell accepted
  `+1` because `[int]` parameter binding normalized it while Bash rejected it.
  Resolution: accept counters as strings, require `^[1-9][0-9]*$`, then convert
  with `System.Numerics.BigInteger`. Follow-up verdict: PASS.
- Quality evaluation cycle 1 found schema type and destination gaps. The
  validators now require the exact contract property set, string `run_id`, a
  nonblank `run_id`, numeric JSON counters, a valid SHA-256, and a directory
  report destination. The cycle 3 isolated evaluator verdict was PASS.

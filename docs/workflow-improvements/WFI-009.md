## Root Cause Hypothesis

The blind cross-model panel is the most expensive verification step in the
workflow (three isolated agents per round), yet it is the FIRST step that
checks two properties that are deterministically checkable beforehand:
(a) whether the sanitized bundle actually contains every artifact the
implementation report declares as its outputs, and (b) whether a
specification-enumerated coverage bar (a finite, listable set of required
elements) is exercised by the delivered test corpus. Because no deterministic
pre-panel check exists for either property, evidence-completeness gaps ride
through bundle assembly and are discovered by the panel itself, converting a
cheap listing comparison into a full panel round plus, in the enumerable-
coverage case, a review-ticket remediation cycle.

## Proposed Change

### WFI Direct Target (project-side)

None. This WFI is a routing record for a plugin-side change (same separation
as WFI-006): the application mechanism is the GitHub Issue lane below.

### GitHub-Issue Lane (plugins/ upstream fix)

| Target File | Change Description |
|---|---|
| plugins/sdd-quality-loop/scripts/prepare-panelist-input.sh (+ prepare-panelist-input.ps1 twin) | After writing the sanitized bundle, verify it contains an artifact section for every path listed in the implementation report's declared-outputs table (path and content hash must match); on any missing or hash-mismatched artifact, fail closed with the list of gaps and do not print an input digest, so no panelist can run against an incomplete bundle. |
| plugins/sdd-quality-loop/skills/cross-model-verify/SKILL.md | Add a deterministic pre-panel readiness step: when the task's specification flags an enumerable coverage requirement (a finite listable set of required elements), require the bundle to include a machine-checkable coverage manifest (required element → exercising fixture) and fail the readiness step when any element is unmapped, BEFORE any panelist is invoked. |

## Expected Effect

Blind-panel rounds returning NEEDS_WORK for evidence-completeness reasons
(bundle missing declared artifacts, or specification-enumerated coverage
gaps) drop from 2 per feature (this period — 2 of 3 first runs; T-002 passed
cleanly) to 0 in the next completed feature with cross-model-enabled tasks,
saving one full
three-panelist round plus up to one review-ticket remediation cycle per
occurrence. The non-decreasing guard is satisfied: no gate, grader, or
threshold is removed or loosened; two fail-closed deterministic checks are
added ahead of an existing expensive step.

## Verification Plan

At the next completed cross-model feature's retrospective, read every
aggregate verdict JSON and panel-related review ticket for the feature and
count panel rounds whose NEEDS_WORK/FAIL findings are evidence-completeness
class (missing bundle artifacts, unexercised specification-enumerated
elements, unwired suites) as opposed to behavioral findings. The proposal
succeeds only if that count is 0 AND the pre-panel readiness step's fail-
closed output (or its clean pass) is recorded in the feature's gate evidence.
Secondary check: no panel round runs against a bundle whose digest was never
preceded by a recorded completeness check.

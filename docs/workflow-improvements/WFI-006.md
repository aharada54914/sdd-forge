## Root Cause Hypothesis

Implementation reports are written as an honest snapshot at their own
authoring commit inside the implementation phase, but neither the report
template nor the quality verification gate's process instructions state that
the report's numbers, statuses, and paths are commit-scoped and may be
superseded by a later same-feature event -- another task's edit touching the
same surface, or a gate-phase artifact normalization such as a path move.
When that later event occurs, nothing in the process reconciles the frozen
report against the gate-time reality; the mismatch is discovered only when an
independent gate reviewer happens to notice and flag it as a finding, which
is why the same class recurs every feature regardless of which model or risk
tier authors the report.

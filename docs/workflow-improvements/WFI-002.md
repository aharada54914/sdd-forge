## Problem Evidence

- GitHub issue #61: spec-review precheck validates absolute-path contracts that include precheck/summary files, while impl-review precheck requires relative paths, specs/-only file sets, and current hashes — no single contract satisfies both.
- `reports/task-review/sdd-forge-mcp/attempt-1/round-3/manual-precheck-note.md`: task-review precheck false-positived on a file-wide Risk/Workflow grep and had to be replaced by a manually executed precheck with a human approval record.
- All 11 quality-gate reports for sdd-forge-mcp (`reports/quality-gate/2026-07-05T021549Z-sdd-forge-mcp-T-*.md`) carry the note "Gate executed manually per the precheck incompatibility recorded in issue #61".
- Retrospective `reports/retrospective/2026-07-05T022733Z-sdd-forge-mcp.md`, FP-001: 4 workflow phases affected in one feature.

## Root Cause Hypothesis

The specification review gate, design review gate, and task decomposition review gate each ship a launch precheck with mutually inconsistent contract expectations (absolute versus relative artifact paths, different required file sets, and different hash-freshness rules), so no invocation contract can satisfy every gate; the quality verification gate inherits the same launch contract. Because the project's workflow files document no sanctioned fallback, every affected run re-derives an ad-hoc manual procedure from scratch, which costs time and risks inconsistent evidence quality between runs.

## Proposed Change

| Target File | Change Description |
|---|---|
| AGENTS.md | Add a "Review gate precheck fallback" subsection documenting the sanctioned manual fallback used in this period: (1) run the precheck steps manually and record results in a `manual-precheck-note.md` inside the round directory, (2) obtain and record explicit human approval of the deviation, (3) reserve reviewer identities in the identity ledger exactly as the automated path would, (4) reference the upstream defect issue (#61) in the note. State that the fallback applies only while the upstream precheck defect is open. |

## Expected Effect

Workflow phases that need an undocumented, re-derived manual fallback drop from 4 per feature (this period) to 0: either the documented fallback is followed as written, or the upstream fix for the review gate plugin (issue #61) makes the fallback unnecessary. Review gate round counts are expected to be unaffected.

# v1.8.0 Release & Documentation Revision Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Revise the user-facing manuals (skill-reference, workflow-guide, README) to the post-design-system reality, then cut the v1.8.0 release (CHANGELOG heading + all version surfaces via `scripts/bump-version.sh`).

**Architecture:** Docs first, release commit last (matching the v1.7.0 pattern where the release commit finalizes all surfaces). Version synchronization is delegated to the existing `scripts/bump-version.sh` (updates 18 plugin manifests, 2 marketplaces, README release line, validate-repository.ps1 assertions, and the release-validation test fixtures in one pass — it only requires the `## v1.8.0` CHANGELOG heading to exist first).

**Tech Stack:** Markdown, Mermaid diagrams, bash release script.

**Branch:** feature/unified-design-system

## Global Constraints

- New version is **1.8.0** (minor: additive design-system integration, no breaking changes). Release date **2026-07-03**.
- `## Unreleased` must remain in CHANGELOG.md as an empty heading above the new `## v1.8.0 (2026-07-03)` heading (release-validation parity test requires the Unreleased heading and exactly one current-release heading — the 0cc7900 lesson).
- Do not edit historical documents: docs/superpowers/plans/*.md and docs/superpowers/specs/*.md are records (their WCAG 2.1 mentions are quoted old-text blocks, correct as history); reports/** are review evidence and must never be touched.
- Do not touch protected files (tests/gates.tests.sh, eval, constant-parity, guard-parity, check-contract*, hooks, reviewer agents, ship SKILL.md). `tests/repository-release-validation.tests.sh` and `tests/validate-repository.ps1` are updated ONLY via `scripts/bump-version.sh`.
- Mermaid edits must not reorder existing nodes/edges; the new loops appear as dotted (`-. label .->`) optional paths, matching the feature's opt-in nature. Do not insert the strings spec-review-loop / impl-review-loop / task-review-loop anywhere new (tests/workflow-documentation.tests.sh asserts their relative order).
- Skill count is now **21** (19 + design-sync-loop + visual-verify-loop); public commands remain 5.
- Local verification limits: `tests/repository-release-validation.tests.sh` and full `tests/validate-repository.ps1` cannot run on this machine (pre-existing python/PS5.1 environment constraints) — the bump script's built-in stale-check plus greps gate locally; CI is authoritative for those two.
- Git commits end with `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`; author `aharada` with dummy email only. All edited docs are UTF-8 Japanese — no encoding/CRLF churn.
- Scope note: AGENTS.md, docs/troubleshooting.md, docs/contributor/* and docs/adr/* were surveyed and contain no stale claims touched by this release (no skill counts, no WCAG versions, no check counts) — they are intentionally out of scope.

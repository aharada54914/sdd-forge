## Cycle 2 (evaluator finding fix)
An independent quality-gate evaluator confirmed a Major defect in
tests/readonly/support/source-scan.ts's `stripComments`: the original
implementation used two non-string-aware regexes
(`/\/\*[\s\S]*?\*\//g` then `/\/\/.*$/gm`) that truncated string literals at
their own embedded `//`. This made the "only scheme://host literal anywhere
in src/ is https://api.github.com" static check (SSRF host-fixing
enforcement, security-spec.md B2) vacuously true — a rogue host literal such
as `"http://attacker.example.com/x"` was silently chopped down to `"http:`
before the host-literal regex ever ran, so the check could never catch a
rogue host. The same truncation also hid same-line code following a
`//`-bearing string literal (e.g. `const u = "sc://y"; writeFileSync(p,d);`
lost everything after the `//`) from the fs-write/child_process/eval checks.

Fix: `stripComments` was rewritten as a small string-aware character scanner
that tracks scan state (normal / single-quote / double-quote /
template-literal / line-comment / block-comment), honors backslash escapes,
and strips characters only while in a comment state — string and template
literal contents (including embedded `//` and `/* */`-shaped text) are now
copied through verbatim. The exported `stripComments(source: string): string`
signature is unchanged.

Three new regression tests were added to
tests/readonly/static-check.test.ts, exercising the evaluator's exact
synthetic reproductions directly against `stripComments`:
  - "Cycle 2: a rogue host literal embedded in a string is preserved (not
    truncated at its own //) and is caught by the host-literal scan"
  - "Cycle 2: same-line code after a string containing // is not swallowed
    as a false comment"
  - "Cycle 2: genuine // and /* */ comments are still stripped (no false
    positive from real comments)"

Total test count grew from 137 (post-T-013 baseline in this checkout, which
already includes T-007's AC-013 contract-conformance suite on top of the
119 recorded at the original T-006 cycle) to 140 with these 3 new cases, all
green. `npx tsc --noEmit` is clean (EXIT:0) and the full `npm test` run
